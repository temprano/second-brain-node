PHASE 01 Set up Node.js on your VPS
STEP 01SSH into your VPS and install Node.js via NVM

NVM (Node Version Manager) is the recommended way — it installs Node in your home directory without needing sudo, and lets you switch versions easily.

# SSH into your VPS first ssh user@your-vps-ip # Install NVM curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.2/install.sh | bash source ~/.bashrc # Install Node.js LTS nvm install --lts node --version # should print v22.x.x or similar npm --version
STEP 02Install PM2 globally

PM2 keeps the webhook server running permanently — it auto-restarts on crash and survives VPS reboots.

npm install pm2@latest -g pm2 --version # confirm installed # Install log rotation (prevents logs filling your disk) pm2 install pm2-logrotate pm2 set pm2-logrotate:max_size 50M pm2 set pm2-logrotate:retain 7
PHASE 02 Upload the pipeline code
STEP 03Upload via SCP or Git

Two options — SCP is quickest for a first deploy, Git is better for ongoing updates.

# Option A: SCP — upload the zip from your Windows machine # Run this in PowerShell on your local machine: scp second_brain_node_search.zip user@your-vps-ip:~/ # Then on the VPS: unzip second_brain_node_search.zip mv second-brain-node ~/second-brain cd ~/second-brain # Option B: Git (recommended for future updates) # Push to GitHub first, then on VPS: git clone https://github.com/yourname/second-brain.git ~/second-brain cd ~/second-brain
💡 Git is better long-term — updating becomes just git pull && pm2 restart second-brain
STEP 04Create your .env on the VPS

Never copy your local .env via SCP — create it fresh on the server.

cd ~/second-brain cp .env.example .env nano .env # or: vim .env

Fill in these values in the .env — the new ones needed on the VPS:

# ── New fields needed for VPS deployment ── # Webhook server port (pick any free port) WEBHOOK_PORT=3579 # Secret key — OpenClaw sends this to authenticate # Generate one: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))" WEBHOOK_SECRET=your-long-random-secret-here # ── Obsidian — IMPORTANT on VPS ────────────────────────────────── # Obsidian runs on your WINDOWS machine, not the VPS # The VPS needs to reach Obsidian over the network # See Phase 3 for how to set this up OBSIDIAN_HOST=YOUR_WINDOWS_IP_OR_TAILSCALE_IP OBSIDIAN_PORT=27124 OBSIDIAN_API_KEY=your-obsidian-api-key # ── All your existing keys ──────────────────────────────────────── ANTHROPIC_API_KEY=sk-ant-... NVIDIA_API_KEY=nvapi-... SERPER_API_KEY=your-serper-key ANKI_HOST=http://YOUR_WINDOWS_IP:8765
⚠ Anki and Obsidian run on your Windows PC, not the VPS. The VPS reaches them over the network — you need your Windows machine's local IP or a Tailscale address. See Phase 3.
STEP 05Install dependencies and create logs folder
cd ~/second-brain npm install mkdir -p logs # Quick sanity check — should print help without errors node src/pipeline/run.js
PHASE 03 Bridge VPS → Obsidian & Anki on Windows
STEP 06Connect VPS to your Windows machine (two options)

The VPS needs to reach Obsidian (port 27124) and Anki (port 8765) on your Windows PC. Two ways to do this:

Option A — Tailscale (recommended, easiest)
Tailscale creates a private encrypted network between your VPS and Windows PC. No firewall rules, no port forwarding. Completely free for personal use.

# On your VPS: curl -fsSL https://tailscale.com/install.sh | sh sudo tailscale up # opens a URL — authenticate with your account tailscale ip # note your VPS tailscale IP (100.x.x.x) # On Windows: download Tailscale from tailscale.com and sign in # Note your Windows PC's Tailscale IP (also 100.x.x.x) # In your VPS .env, use the Windows Tailscale IP: OBSIDIAN_HOST=100.x.x.x # your Windows Tailscale IP ANKI_HOST=http://100.x.x.x:8765

Option B — Local network IP (simpler if VPS and PC are on same network)

# On Windows, find your local IP: # Open cmd → ipconfig → look for IPv4 Address (e.g. 192.168.1.50) # In your VPS .env: OBSIDIAN_HOST=192.168.1.50 ANKI_HOST=http://192.168.1.50:8765 # On Windows Firewall: allow inbound on ports 27124 and 8765 # Control Panel → Windows Firewall → Advanced → Inbound Rules → New Rule
💡 Tailscale is the better choice — it works even if your PC's IP changes, requires no firewall rules, and encrypts everything automatically.
STEP 07Update Obsidian Local REST API binding

By default Obsidian's REST API only accepts connections from the same machine (127.0.0.1). You need to allow external connections from your VPS.

  • Obsidian → Settings → Local REST API → click gear ⚙
  • Enable Show advanced settings
  • Change Binding Host from 127.0.0.1 to 0.0.0.0
  • Click Re-generate Certificates
  • Restart Obsidian
⚠ Setting binding to 0.0.0.0 exposes the API to your network. If using Tailscale this is safe — only Tailscale devices can reach it. If using a public IP, add firewall rules to restrict access.
PHASE 04 Start the webhook server with PM2
STEP 08Start with PM2 and enable auto-restart on reboot
cd ~/second-brain # Start the webhook server pm2 start ecosystem.config.cjs # Check it's running pm2 list pm2 logs second-brain --lines 20 # Register as system service (auto-starts on VPS reboot) pm2 startup # ↑ This prints a command — copy and run it exactly as shown # It will look like: sudo env PATH=$PATH:/home/user/.nvm/... pm2 startup systemd ... # Save current process list pm2 save # ← required — without this, apps won't restart after reboot
✓ Both pm2 startup AND pm2 save are required. Missing either means the app won't auto-restart after reboot. Run both every time you add a new app.
STEP 09Test the webhook server
# Health check (no auth needed) curl http://localhost:3579/health # Expected response: { "status": "ok", "uptime": 42, "jobs": 0, "version": "1.0.0" } # Test a real job (replace YOUR_SECRET with your WEBHOOK_SECRET) curl -X POST http://localhost:3579/run \ -H "X-API-Key: YOUR_SECRET" \ -H "Content-Type: application/json" \ -d '{"command": "research: best VPS providers 2025"}' # Expected response: { "status": "queued", "jobId": "a1b2c3d4", "intent": "Research", "topic": "best VPS providers 2025", "deck": "Research::Best Vps Providers 2025", "folder": "100-Learning/Research" } # Check job status curl -H "X-API-Key: YOUR_SECRET" http://localhost:3579/status/a1b2c3d4
PHASE 05 Connect OpenClaw to the webhook
STEP 10How to trigger the pipeline from OpenClaw

Give OpenClaw a tool or action that POST requests to your webhook. The bot detects the intent: topic pattern and sends it to the pipeline.

The command format:

You say to OpenClawObsidian folderAnki deck
research: best home owners insurance right now 100-Learning/Research Research::Best Home Owners...
learning: Remotion full review 100-Learning/Topics Learning::Remotion Full Review
study: quantum entanglement 100-Learning/Study Study::Quantum Entanglement
review: Make It Stick book 100-Learning/Reviews Reviews::Make It Stick Book
news: AI regulation EU latest 100-Learning/News News::Ai Regulation Eu Latest

The OpenClaw tool/action to add:
Add a webhook action in OpenClaw that sends a POST request when it detects the pattern. The payload is just JSON with a "command" field.

# Example: OpenClaw webhook action config # URL: http://your-vps-ip:3579/run # Method: POST # Headers: X-API-Key: your-webhook-secret # Content-Type: application/json # Body: { "command": "{{user_message}}" } # OpenClaw trigger phrase examples (configure in bot settings): # Starts with "research:" # Starts with "learning:" # Starts with "study:" # Starts with "review:" # Starts with "news:"
💡 The webhook responds in under 100ms with a jobId — the pipeline runs async in the background. OpenClaw can immediately confirm "Queued! Researching 'best home owners insurance'..." without waiting for the full pipeline to complete.
